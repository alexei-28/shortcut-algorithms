#### Массивы:
#### 1. Вставка элемента в конец массива
```
int[] arr = {1, 2, 3, 4, 5};
// или ArrayList<Integer> arr = new ArrayList<>(Arrays.asList(1, 2, 3, 4, 5));
arr.add(6);
```
#### Ответ: arr.append (6) — O(1) амортизированная. 
В большинстве случаев это просто добавление в конец. Иногда требуется расширение массива — O(n), но это происходит редко.

#### 2. Обращение к элементу массива по индексу
```
arr[0]
```
#### Ответ: arr[0] — O(1)
Прямой доступ по индексу.

#### 3. Вставка элемента в начало массива
```
arr.insert(0, 0);
```
#### Ответ: arr.insert (0, 0) — O(n)
Нужно сдвинуть все элементы вправо.

#### 4. Проверка на наличие элемента в массиве
```
arr.contains(5);
```
#### Ответ: 5 in arr — O(n)
Линейный поиск по массиву.

---
#### Строки:
#### 5. Проверка длины строки
```
   String s = "hello world";
   s.length();
```   
#### Ответ: len(s) — O(1)
Длина строки хранится отдельно

#### 6. Доступ к элементу строки по индексу
```
   s.charAt(0);
```
#### Ответ: s[0] — O(1)
Прямой доступ по индексу

#### 7. Добавление символа в конец строки
```
   s + "!";
```
#### Ответ: s + «!»— O(n)
Нужно создать новую строку и скопировать все символы

#### 8. Проверка наличия подстроки в строке
```
   s.contains("world");
```
#### Ответ: «world» in s — O(n*m),
где n = длина строки, m = длина подстроки.

---
#### Связный список (односвязный):

#### 9. Получение значения первого узла
```
// head указывает на начало списка: 1 → 2 → 3 → null
head.value;
```
#### Ответ: head. value — O(1)
Прямой доступ к значению узла

#### 10. Получение ссылки на второй узел
```
head.next;
```
#### Ответ: Поиск значения 3 — O(n)
Нужно пройти от начала до конца списка

#### 11. Получение значения третьего узла в связном списке
```
head.next.next.value;
```
#### Ответ: O(n)

### Задание 2
Для каждой задачи выберите подходящую структуру данных (массив или связный список) и объясните выбор:

#### Пример 1: 
Нужно хранить список задач, куда постоянно добавляются новые задачи в конец, и иногда нужно посмотреть задачу по номеру.
#### Ответ: Массив (динамический)
Почему: Добавление в конец — O(1) амортизированная для массива, а доступ по номеру — O(1) против O(n) в связном списке

#### Пример 2: 
Нужно реализовать очередь сообщений, где сообщения часто добавляются в конец и удаляются из начала.
#### Ответ: Связный список
Почему: Частое удаление из начала — O(1) для связного списка, но O(n) для массива (нужно сдвигать все элементы)

#### Пример 3:
Нужно хранить данные о погоде за год, где часто требуется получить данные за конкретный день по его номеру.
#### Ответ: Массив (фиксированный или динамический)
Почему: Доступ по индексу (номеру дня) — O(1) для массива против O(n) в связном списке
